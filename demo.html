Flexgrid - a simple grid based on flexbox layout

No more floats! Regular grid layout is based on both block and inline flow directions, however, the flexbox (or just flex) layout is based on "flex-flow directions". Thus float and clear have no effect. Using float causes the display-property to compute to block. Using flexbox also means less JavaScript! Everything is, of course, responsive and stacks perfectly. Try by resizing your browser or view this site from a mobile device or tablet.

Flexgrid is open-source, and everything can easily be modified. Let's say you want more or less than the standard 12 columns. Because Flexgrid is made with SCSS, you only have to edit a single number, and then recompile. The entire sourcecode is available on GitHub, where you're also welcome to fork the project.

Please note: flexbox is only partially supported by IE10. Any IE-version lower than 10 does not support flexbox. However, Flexgrid has many fallbacks included, so IE11 should be safe. More info here.

Download

Packages

If you want to download Flexgrid using either NPM, or Yarn.

  • npm install flexgrid.co
  • yarn install flexgrid.co

Tailwind CSS

Inspired by the Tailwind CSS-syntax, there's also a Tailwind version of Flexgrid.

To use it, when you compile or import Flexgrid, set the $enable-tailwind-variable to true. You can do this either by editing _variables.scss, or overriding it in your own variables, as the variable has the !default-flag attached.

If you don't know what Tailwind CSS is, or if you don't want to use it, simply ignore this section as it's something you have to opt in to.

.container

Flexgrid comes with responsive containers that boxes the content in which are controlled by @media-queries. These @media-queries trigger at the same widths defined by the Bootstrap-framework. However, as a small detail, Flexgrid bases its width-triggers on rem instead of px. This creates a much more fluid layout. While em is relative to the font-size of its direct or nearest parent, rem is only relative to the html (root) font-size. Flexgrid font-size is set to 16.

There is also a fluid container that has no @media-queries. Look further down for examples.

Columns

Flexgrid comes with columns with percent-based widths allow fluid resizing of columns and rows.

Flexgrid comes with 4 layout classes:

  • xs (xtra small - for mobile devices and small tablets)
  • sm (small - for large mobile devices and tablets)
  • md (medium - for tablets and very small laptops)
  • lg (large - for smaller laptops and -desktops)
  • xl (xlarge - for desktops)

These classes can be combined to create dynamic and flexible layouts.

Pro tip: each class scales up, so if you wish to set the same widths for 'sm' and 'md', you only need to specify 'sm'.

Auto width

This defines the ability for a flex item to grow if necessary. It accepts a unitless value that serves as a proportion. It dictates what amount of the available space inside the flex container the item should take up.

Justifying content: start, center, end

start: (default): items are packed toward the start line.
center: tems are centered along the line.
end: items are packed toward to end line.

Pro tip: You can combine <prefix>-center and <prefix>-middle to perfectly center content on a page. No more display: table; on the outer and display: table-cell; vertical-align: middle; on the inner element, or even position: absolute; translate: transformX(-50%); top: 50%; to vertically and horizontally center content.
